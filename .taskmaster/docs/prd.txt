**GoTo Optical — Patient-Attribution & Satisfaction Capture Web App**
*Single-Page Product Requirements Document (PRD)*

---

### 1 . Purpose & Vision

Extend **GoTo Optical's** marketing website with a lightweight, HIPAA-ready web app that:

1. **Captures how every patient discovered the practice** (walk-in, Meta/Facebook ads, Instagram ads, WhatsApp Business ads, other).
2. **Records their vision-insurance provider**—pre-filled from the 15+ plans already shown on the public site (e.g., VSP, EyeMed, Spectera, United Healthcare). ([GoTo Optical and Eye Care][1])
3. **Collects a 1-click, 1–5-star satisfaction score for the attending doctor** (e.g., Dr. Bruce Goldstick, Dr. Joby). ([GoTo Optical and Eye Care][2])
4. **Surfaces real-time dashboards** linking acquisition channel → insurance mix → doctor rating, so the owner can optimize ad spend and clinical quality.

Outcome: raise booked eye-exam conversions, justify marketing ROI, and keep Google rating ≥ 4.9 ⭐️.

---

### 2 . Problem Statement

Current booking widget records appointments but **doesn't tie visits back to ad channels, insurance, or doctor satisfaction**, forcing manual Excel work to answer simple questions like "Did July's Instagram campaign bring higher-value VSP patients?" or "Which doctor needs coaching?".

---

### 5 . User Journey (Happy Path)

1. **Arrival / Booking**

   * Walk-in: Lia hands patient an iPad with intake link.
   * Online ad click: UTM parameters auto-prefill "Meta-Ads → Dallas-Moms-Jul-2025".
2. **Smart Intake Form** (≤ 30 s)

   * "How did you hear about us?" (radio + hidden UTM fallback)
   * Insurance provider dropdown (logos for VSP, EyeMed, … ) ([GoTo Optical and Eye Care][1])
   * Basic demographics + consent checkbox
3. **Visit completed** — doctor selected in EHR (Goldstick / Joby / Any).
4. **Instant SMS or email** — 1-click 5-star prompt + free-text comment.
5. **Admin Dashboard** — real-time funnel: Channel → Insurance → Doctor Rating.

### 6 . Functional Requirements

| #  | Feature                              | Details                                                                            |
| -- | ------------------------------------ | ---------------------------------------------------------------------------------- |
| F1 | **Responsive Intake Form**           | English & Spanish, validates required fields; offline-cache PWA.                   |
| F2 | **Attribution Auto-capture**         | Parse UTM/FBCLID; default to "Walk-in" if none.                                    |
| F3 | **Insurance Selector**               | Pre-loaded list of accepted plans (VSP-EyeMed-Spectera-UHC-…); free-text override. |
| F4 | **Doctor Satisfaction Survey**       | 1–5 ⭐️ + optional comment; triggered post-visit via SMS/email or kiosk.            |
| F5 | **Real-Time Analytics**              | Channel mix, insurance mix, average rating per doctor; CSV & webhook export.       |
| F6 | **Role-Based Access**                | Front-Desk (create), Marketing (read dashboards), Provider (view own scores).      |
| F7 | **Scheduling Integration (Phase 2)** | Pass form data to existing ScheduleYourExam flow. ([GoTo Optical and Eye Care][3]) |

### 7 . Non-Functional Requirements

| Category                  | Requirement                                                                   |
| ------------------------- | ----------------------------------------------------------------------------- |
| **Performance**           | p95 form submission processing < 300 ms                                       |
| **Availability**          | 99.9 % uptime                                                                 |
| **Scalability**           | 15 k submissions / day without degradation                                    |
| **Security & Compliance** | HIPAA, SOC 2, GDPR; AES-256 at rest, TLS 1.3 in transit                       |
| **Accessibility**         | WCAG 2.2 AA (color-blind safe, screen-reader labels)                          |

### 9 . Tech Stack (recommended)

| Layer     | Choice                                  | Rationale                                         |
| --------- | --------------------------------------- | ------------------------------------------------- |
| Front End | **Next.js 14 + Tailwind**               | Consistent with existing GoTo Optical stack reuse |
| Back End  | **FastAPI (Python) + PostgreSQL**       | Async, JSON-friendly, HIPAA-ready                 |
| Auth      | **Auth0 HIPAA BAA**                     | SSO + granular roles                              |
| Hosting   | **Azure App Service** or **Vercel Pro** | CI/CD; regional redundancy                        |
| Analytics | **Supabase Realtime** or **Redash**     | Lightweight dashboarding                          |

### 11 . Risks & Mitigations

| Risk                                | Impact               | Mitigation                                         |
| ----------------------------------- | -------------------- | -------------------------------------------------- |
| Low survey response                 | Poor doctor insights | Kiosk prompt before patient leaves; reminder SMS   |
| Incorrect attribution (ad blockers) | ROI misreporting     | Fallback manual question + periodic data audits    |
| HIPAA breach                        | Legal & financial    | Encryption, audit logging, yearly compliance audit |
